## ðŸŽ¯ Key Design Decisions

### 1. Copilot Instructions Organization

**Why path-specific > language-specific?**
- More precise targeting
- Avoids conflicts with general JS patterns
- Enforces userscript-specific security rules

### 2. IIFE Pattern

**Why not ES modules?**
- Userscripts run in page context
- Avoid global namespace pollution
- Compatible with all userscript managers

### 3. Debounced MutationObserver

**Why not polling?**
- Better performance (O(changes) vs O(time))
- Lower CPU usage
- Responds immediately to DOM changes

### 4. @grant none

**Why avoid GM_* functions?**
- More secure (runs in page context)
- Simpler code
- Better compatibility
- Use only when necessary (storage, XHR)

### 5. JSDoc Over TypeScript

**Why not compile from TypeScript?**
- Simpler build process
- Direct .user.js installation
- Good enough type checking with @ts-check
- Can use TypeScript for complex projects
